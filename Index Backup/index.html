④ Reservations & Savings Plans

How discount instruments change and where optimisation opportunities exist.

🏛️ Under EA (Today)

  • Reservation scope: Enrollment-wide (shared) or single Subscription
  • Savings Plan scope: Enrollment or Subscription
  • Benefits apply based on manual scoping selection
  • Sub-scoped RIs only benefit that one subscription
  • Monthly reservations paid in local currency

📄 Under MCA (Future)

  • Reservation scope: Billing Profile level — all subs covered
  • Savings Plan scope: Billing Profile level — broadest default
  • Sharing is the default — no manual scoping needed
  • Monthly non-USD reservations canceled during migration — repurchase after
  • Cancellations don't count toward $50K rolling 12-month cap

💡 Optimisation Opportunity

With MCA, Reservations and Savings Plans are automatically shared at the Billing Profile level — broader coverage, better utilisation, less manual scoping.

⑤ CAPT (Customer Affiliate Purchase Terms)

Understanding how CAPT enrollments differ from standard Billing Profiles in the MCA world.

🏗️ Where CAPT Fits in the MCA Hierarchy

🏦 Billing Account (Parent MCA)
↓
🧾 Billing Profile (Standard)
↓
📋 Invoice Sections & Subs
🔗 Child MCA Billing Account (CAPT)
↓
🧾 Own Billing Profile
↓
📋 Own Invoice Sections & Subs

🧾 Standard Billing Profile

  • Created from: A standard (non-CAPT) EA Enrollment
  • Location: Sits directly under the parent Billing Account
  • Management: Customer self-manages billing, invoices, and cost views
  • Invoice: Customer receives and pays the invoice directly
  • RIs & SPs: Shared at the Billing Profile level across all subscriptions
  • Hierarchy depth: Billing Account → Billing Profile → Invoice Sections → Subscriptions
  • Autonomy: Full control — customer assigns all billing roles

🔗 CAPT — Child MCA Billing Account

  • Created from: A CAPT (Affiliate) EA Enrollment
  • Location: Creates a separate child Billing Account under the parent
  • Management: Affiliate manages billing on behalf of the customer
  • Invoice: Affiliate handles invoicing and may apply markups or bundle services
  • RIs & SPs: Shared within the child account's own Billing Profile
  • Hierarchy depth: Parent Billing Account → Child Billing Account → Billing Profile → Invoice Sections → Subscriptions
  • Autonomy: Affiliate controls billing; customer retains Azure resource access
Aspect🧾 Standard Billing Profile🔗 CAPT (Child MCA)
SourceStandard EA EnrollmentCAPT (Affiliate) EA Enrollment
MCA Entity CreatedBilling Profile under parent Billing AccountSeparate child Billing Account with its own Billing Profile

💡 Key Takeaway

CAPT enrollments become their own billing entity — a child MCA Billing Account — rather than just a Billing Profile. This preserves the affiliate billing relationship while keeping the customer's Azure resources and RBAC completely intact.

⑥ What Changes — Action Items

Comprehensive list of changes after EA → MCA migration. Source: Microsoft Learn — Changes after migration

🏦 Remittance Details

MCA bank/wire details differ from EA. AP needs separate records for final EA invoice and new MCA invoices.

Action: Update wire transfer details using remittance info at the bottom of your MCA invoice.

🧾 Dual Invoices in Transition

You may receive two invoices (one EA, one MCA) in the transition month. MCA invoice covers a calendar month and is generated on the 5th–7th of the following month.

Action: Alert AP team. Add accounts payable email as invoice recipient on MCA billing profile.

📊 Cost Views & Budgets

Custom views, shared views, and budgets do not migrate to MCA. They must be recreated in the new billing scope.

Action: Document existing views/budgets before migration. Recreate under MCA billing scope.

📈 Power BI Integration

The Azure Cost Management connector scope changes. Use Billing Profile ID (14-char) instead of EA enrollment number.

Action: Update Power BI connector scope. Rebuild any enrollment-scoped reports.

🔌 API Endpoints

EA API keys are retired. EA API calls do not work with MCA. You must migrate to Microsoft Cost Management APIs (OAuth2 + service principal).

Action: Migrate scripts using the EA-to-MCA API migration guide.

📄 Azure Usage File Changes

MCA usage/charges file has more columns and different naming conventions compared to EA. Field names and structure differ.

Action: Review field changes and update any scripts that parse the usage CSV.

🎫 Reservations (RIs)

Monthly non-USD reservations are canceled during migration at next payment. RI charges may appear under the primary invoice section — verify placement.

Action: Repurchase after migration. Verify RI charges land in the correct invoice section; submit transfer request if needed.

💎 Savings Plans

Savings Plans purchased in a non-USD currency are canceled during migration.

Action: Repurchase Savings Plans after migration completes.

📦 Export Jobs

EA export jobs do not auto-migrate to MCA. You must manually recreate them under MCA billing scope.

Action: Reconfigure exports using Azure Commerce Toolbox or PowerShell. Confirm MCA billing scope alignment.

📜 Tax Exemption Certificates

Existing EA tax exemptions are not auto-applied to MCA. You need to re-associate via support request.

Action: Create an Azure support request to associate your tax exemption certificate to the new MCA account.

📂 Historical Data

Cost data before migration stays in the EA scope — it does not move to MCA. Account owners/subscription owners lose access to EA billing scope data.

Action: Download cost/usage data and invoices before transferring subscriptions. Switch scopes to view historical EA data.

🔄 Automatic Purchases

Any automatic purchases set up under the old EA enrollment do not carry over.

Action: Re-configure automatic purchases under the new MCA.

🏗️ Terraform / IaC Subscription Vending

Subscription vending modules (e.g. Azure/lz-vending/azurerm) require parameter changes for MCA billing scope.

Action: Update Terraform modules and IaC pipelines to reference MCA billing account / profile IDs.

📋 Management Groups in Cost Management

Subscriptions in management groups under MCA are not yet supported in Cost Management. Cost views, APIs, and automation scripts scoped to management groups need updating.

Action: Rebuild dashboards using Billing Profile ID or Invoice Section ID. Update API endpoints and automation scripts.

🌐 Third-Party Cost Tools

Tools like CloudHealth, Cloudability, etc. need to be updated to pull MCA cost data instead of EA data. Most providers have documented migration processes.

Action: Contact your third-party provider to update the integration to MCA billing scope.

🌱 Emissions Impact Dashboard

You must update the Billing Account ID in Power BI with a Billing Account Administrator role (Reader/Contributor/Owner).

Action: Update Billing Account ID in the Emissions Impact Dashboard connector.

🛒 Marketplace

Marketplace purchases can now be configured to appear on the same Azure invoice or a separate invoice — giving more billing flexibility than EA.

Action: Configure Marketplace invoice preference in the billing profile settings.

⑧ Why Customers Should Move to MCA

The Microsoft Customer Agreement replaces the Enterprise Agreement with a modern, digital-first billing experience. Here's why it matters.

📄 What is MCA?

A simplified purchase agreement — presented, accepted, and stored through a fully digital experience. It's short, doesn't expire, and automatically updates as you add products. No more annual renewals or renegotiations.

📝 Never Expires

Unlike EA which requires renewal every 1-3 years with a new enrollment number, MCA is signed once digitally and never expires. Your Billing Profile IDs stay stable forever — no more broken dashboards or scripts at renewal.

💳 Flexible Invoicing

Create multiple billing profiles for separate invoices per business unit, project, or region. Configure Marketplace purchases on the same invoice or separate. No more splitting one giant EA invoice manually.

📊 Real-Time Cost Management

Enterprise-grade cost analysis, budgets, and savings recommendations built into the Azure portal. Set spending alerts per billing profile or invoice section. Integrated Power BI with modern OAuth2 APIs.

🌐 Multi-Tenant Billing

Natively manage billing across multiple Azure AD tenants from a single MCA billing account. No need to guest users into a primary tenant — associate tenants securely while maintaining control.

🔒 Granular RBAC for Billing

Separate billing roles from Azure resource roles. Assign Owner, Contributor, Reader, Invoice Manager at billing account, profile, or section level. More precise access control than EA's flat role model.

⚡ Instant Purchasing

No upfront costs required. Pay-as-you-go billing for most services. Buy what you need when you need it. Negotiate commitments (MACC) when ready. Start using Azure immediately after signing.

🧩 Simpler Hierarchy

The Account layer is removed. Subscriptions attach directly to Invoice Sections (formerly Departments). Account Owners become Subscription Creators. Fewer layers = easier governance.

🎫 Reservation Sharing

Reserved Instances and Savings Plans auto-share at the Billing Profile level across all subscriptions. More workloads benefit from existing commitments without manual scoping per account.

🛡️ Zero-Impact Migration

This is a billing change, not a cloud migration. No resources move. No downtime. No permission changes. Same Azure portal, same RBAC, same running services. Only the billing wrapper updates.
